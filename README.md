# Poker Legends 🃏

友人同士でオンライン対戦できるテキサスホールデム・キャッシュゲームのWebアプリです。
各自のスマホ/PCのブラウザから参加でき、カード配布・役判定・ポット計算・ポイント記録まですべて自動です。

## 機能

- ノーリミット・テキサスホールデム(2〜9人)
- ルームコード(4文字)で簡単に合流。「招待」ボタンのQRコードを読み取れば、コード入力済みの状態でサイトが開く
- ブラインド額・初期持ち点・持ち点上限(バイインMax)はルーム作成時に自由設定
- キャッシュゲーム仕様: いつでもチップ追加(リバイ)可能(上限まで)、ポイント(±)を常時表示
- 「戻す」機能: 初期持ち点を超えた勝ち分をテーブルから下ろしてポイントを確定(以降のハンドで失わない)
- レイズ額は −/+ ボタン(BB単位)・スライダー・プリセット(最小/½ポット/ポット/All-in)で素早く設定
- オールイン・サイドポット対応
- 45秒の考慮時間タイマー(時間切れは自動チェック/フォールド)
- 回線が切れてもページを開き直せば同じ席に自動復帰
- スマホ最適化UI

## ローカルで起動

```bash
npm install
npm start          # http://localhost:3000
```

ロジックのテスト:

```bash
npm test           # ランダム対戦シミュレーション + 役判定テスト
```

## 遊び方

1. ホストがアプリを開き、名前・ブラインド・持ち点を入れて「ルーム作成」
2. 「招待」ボタンのQRコードを友だちに見せる(またはURL/4文字コードを共有)
3. 友だちは同じURLを開いて名前とコードを入れて「参加」
4. 全員揃ったらホストが「ゲーム開始」
5. ハンドは自動で次々に進行。負けたら「チップ追加」でリバイ
6. 終了時は各自の「ポイント」を見て整理

## インターネットに公開する方法

### 方法A: Render にデプロイ(無料・常設、おすすめ)

1. このフォルダを GitHub リポジトリにする
2. [render.com](https://render.com) → New → Web Service → リポジトリを選択
3. Build Command: `npm install` / Start Command: `npm start` で作成
4. 発行された URL(`https://xxx.onrender.com`)を友だちに共有するだけ

Railway や Fly.io でも同様にデプロイできます(Node.js が動けばOK。`PORT` 環境変数に対応済み)。

### 方法B: 自分のPCを一時的に公開(インストール不要・即席)

サーバーをローカルで起動したまま、別ターミナルで:

```bash
~/.local/bin/cloudflared tunnel --url http://localhost:3000
```

ログに表示される `https://～.trycloudflare.com` のURLを友だちに共有します(遊んでいる間だけPCとこのコマンドを起動しておく必要があります。URLは起動のたびに変わります)。
cloudflared 未導入なら `npx localtunnel --port 3000` でも可(訪問者に確認画面が出ます)。

### 方法C: 同じWi-Fi内なら

同じネットワークにいる場合は `http://<あなたのPCのIPアドレス>:3000` を開くだけで参加できます。

## 構成

```
server.js          # Express + Socket.IO サーバー(ルーム管理)
game/table.js      # ゲーム進行エンジン(ベッティング、ポット、サイドポット)
game/evaluator.js  # 役判定(7枚から最強の5枚を評価)
public/            # クライアント(ロビー + ゲーム画面)
test/sim.js        # シミュレーションテスト
```

ゲーム状態はすべてサーバー側で管理しており、他人の手札がクライアントに送信されることはありません。
